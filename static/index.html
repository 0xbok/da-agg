Hello! Store data on Avail, EigenDA or NearDA. Please bear with some guiding instructions.
  • Blob IDs are stored in an in-memory object on the backend for quick iteration.

Store Blob

Enter any message in data and click Store after choosing the DA layer.
  • EigenDA will quickly respond due to async processing.
  • Avail and Near request will take time as it'll wait for it to be included in a block.

Get Blob Status

Store will respond back with an application-generated ID. Copy the ID using Copy button above, use it to get blob status.
Application automatically routes the request to the correct DA layer.
  • EigenDA will show "PROCESSING" for some time before finalizing.
  • Avail and Near shows finalized because Store responds only when data is stored.
  • Once status is "FINALIZED", move to the next API.

Get Blob Data

Enter the same ID to get blob data.
  • If it's not finalized, it will return error or fail silently.
